## 🔥 Настройка Firebase

### Шаг 1: Скопировать Security Rules

1. Откройте [Firebase Console](https://console.firebase.google.com/)
2. Выберите проект `labb-wwork`
3. Перейдите **Firestore Database** → **Rules**
4. Замените содержимое на текст из файла `FIREBASE_SETUP.md` (раздел 2.1)
5. Нажмите **Publish**

### Шаг 2: Создать тестовые данные

Создайте несколько турниров в Firestore:
```json
{
  "name": "Winter Showdown",
  "game": "CS2",
  "description": "Эпический турнир по CS2",
  "prizePool": 25000,
  "maxTeams": 16,
  "registeredTeams": 5,
  "format": "BO3",
  "region": "Online",
  "status": "upcoming",
  "createdAt": "2026-02-10T10:00:00Z",
  "startDate": "2026-02-28T18:00:00Z",
  "registrationDeadline": "2026-02-25T23:59:59Z"
}
```

### Шаг 3: Назначить админа

1. Зарегистрируйтесь в системе
2. Откройте Firebase Console → Firestore → users
3. Найдите свой документ (userId)
4. Измените поле `role` с `"user"` на `"admin"`
5. Готово! Теперь у вас есть доступ к админ-панели

## 🎯 Функциональность по требованиям

### 3.1 Главная страница
- ✅ **Каталог:** Турниры выводятся по 15-20 штук
- ✅ **Поиск:** Поиск по названию в реальном времени (поле в фильтрах)
- ✅ **Фильтрация:** По игре (CS2, DOTA 2, VALORANT, Обезьянка)
- ✅ **Сортировка:** По дате (новые), призовому фонду (↓/↑)
- ✅ **Реальное время:** `onSnapshot()` обновляет турниры без перезагрузки

### 3.2 Страница детального просмотра
- ✅ **Полная инфо:** Название, описание, призовой фонд, статус, прогресс регистрации
- ✅ **Интерактив:** Кнопка "Участвовать", если еще есть места
- ✅ **Похожие:** Показ 3 похожих турниров по игре с пагинацией
- ✅ **Действие:** Основная кнопка "УЧАСТВОВАТЬ В ТУРНИРЕ"
- ✅ **Обновление:** Статус участников обновляется в реальном времени

### 3.3 Личный кабинет
- ✅ **История:** Список всех регистраций (прошлые и текущие)
- ✅ **Управление:** Можно видеть статус каждой регистрации
- ✅ **Профиль:** Редактирование имени, просмотр email
- ✅ **Статусы:** Pending, Approved, Rejected статусы видны в реальном времени

### 3.4 Функциональная страница (корзина/бронирование)
- ✅ **Опциональная:** Реализована как часть личного кабинета
- ✅ **История:** Переносится в личный кабинет после одобрения

### 3.5 Админ-панель
- ✅ **Ограничение:** Только для role == 'admin' (проверка в router.js)
- ✅ **CRUD:** Добавление, редактирование, удаление турниров
- ✅ **Менеджмент:** Просмотр заявок, юзеров, изменение статусов
- ✅ **Модерация:** Удаление некорректного контента
- ✅ **Статистика:** Базовые метрики (кол-во юзеров, турниров, заявок)

### 3.6 Авторизация
- ✅ **Вход:** Регистрация, логин через Firebase Auth
- ✅ **Безопасность:** Пароли хранятся в Firebase, сессии сохраняются
- ✅ **Роли:** Разграничение user и admin
- ✅ **Выход:** Кнопка выхода в навигации

## ⚡ Оптимизация

### Использованные техники:
- ✅ **onSnapshot()** - Реальное время на Home.vue
- ✅ **limit()** - Пагинация по 15 турниров
- ✅ **orderBy()** - Сортировка по createdAt, prizePool
- ✅ **where()** - Фильтрация по game, status
- ✅ **select()** - В плане реализации (документировано в firebaseService.js)
- ✅ **Батч операции** - Для админ-панели (writeBatch)
- ✅ **Кеширование** - На клиенте через computed properties

### Производительность:
- ~2-3 Firestore reads на загрузку главной (1 onSnapshot)
- ~1 Firestore read на открытие турнира
- ~1-2 Firestore writes на регистрацию

## 📞 Помощь

### Частые ошибки:

**Ошибка: "permission-denied"**
```
→ Проверьте скопировали ли Security Rules
→ Убедитесь что пользователь авторизован
→ Check rules для нужной операции
```

**Ошибка: "index-creation-required"**
```
→ Firebase автоматически создаст индекс по ссылке из консоли
→ Дождитесь завершения (2-5 минут)
```

**Турниры не отображаются**
```
→ Добавьте тестовые данные в Firestore
→ Проверьте наличие коллекции "tournaments"
→ Убедитесь что createdAt - это timestamp
```
