# Отчет по тестированию проекта CYBERTOUR

## Цель тестирования

Целью тестирования была проверка работоспособности Vue-приложения, компонента карточки турнира и процесса регистрации пользователя на турнир. Также была добавлена автоматическая проверка проекта в GitHub Actions, чтобы тесты запускались не только локально, но и в самом GitHub при отправке изменений.

## Что было добавлено

- Настроен фреймворк unit-тестирования Vitest.
- Добавлена библиотека `@vue/test-utils` для тестирования Vue-компонентов.
- Добавлена среда `jsdom`, чтобы компонент можно было тестировать в имитации браузера.
- В `package.json` добавлены команды:
  - `npm test` - запуск unit-тестов;
  - `npm run test:watch` - запуск тестов в режиме наблюдения.
- Добавлен GitHub Actions workflow `.github/workflows/tests.yml`.

## Автоматические тесты в GitHub

Теперь при каждом `push` или `pull_request` в ветку `main` GitHub автоматически выполняет:

1. Установку зависимостей через `npm ci`.
2. Запуск unit-тестов через `npm test`.
3. Проверку production-сборки через `npm run build`.

Если один из шагов завершится ошибкой, GitHub покажет красный статус проверки.

## Метод тестирования

Использовалось тестирование методом серого ящика.

Серый ящик означает, что тестирование выполнялось с частичным знанием внутренней структуры приложения. Проверялось не только то, что видит пользователь в интерфейсе, но и внутренняя логика компонента:

- какие props получает компонент;
- когда показывается кнопка регистрации;
- когда показывается ссылка на вход;
- блокируется ли регистрация при заполненном турнире;
- вызываются ли функции Firebase при успешной регистрации;
- отправляется ли событие `registered` после регистрации.

## Unit-тесты

Был добавлен файл:

`src/components/__tests__/TournamentCard.test.js`

Проверки:

1. Компонент отображает игру, название турнира, призовой фонд и ссылку на страницу турнира.
2. Для неавторизованного пользователя показывается ссылка на вход вместо кнопки регистрации.
3. Если турнир заполнен, кнопка регистрации становится неактивной.
4. При успешной регистрации вызываются функции Firebase и компонент отправляет событие `registered`.

## Найденные проблемы

Во время настройки тестирования были выявлены следующие проблемы:

1. В проекте изначально не было настроено unit-тестирование.
2. В `package.json` отсутствовала команда `npm test`.
3. В проекте не было автоматического запуска тестов через GitHub Actions.
4. `npm audit` обнаружил уязвимости в зависимостях:
   - 2 moderate;
   - 3 high;
   - 1 critical.
5. При написании тестов Firebase нельзя было вызывать напрямую, потому что тесты не должны обращаться к реальной базе данных.
6. Формат отображения призового фонда зависел от локали: вместо `$5,000` компонент выводил `$5 000`.

## Что было исправлено

1. Добавлены зависимости для тестирования:
   - `vitest`;
   - `@vue/test-utils`;
   - `jsdom`.
2. Добавлены команды запуска тестов в `package.json`.
3. Добавлена настройка `test.environment = 'jsdom'` в `vite.config.js`.
4. Для Firebase в тестах сделаны mock-объекты, чтобы тесты не работали с реальной базой данных.
5. Проверка призового фонда в тесте сделана устойчивой к разному форматированию пробелов.
6. Выполнена команда `npm audit fix`, после чего количество уязвимостей стало равно 0.
7. Добавлен GitHub Actions workflow для автоматического запуска проверок.
